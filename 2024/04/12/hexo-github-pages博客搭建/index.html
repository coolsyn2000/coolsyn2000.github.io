Hexo-Icarus & Github-Pages博客搭建

环境

  • 安装node.js

    Node.js 安装包及源码下载地址为：https://nodejs.org/en/download

  • 安装git

    Git 各平台安装包下载地址为：http://git-scm.com/downloads

  • 安装hexo

    npm install -g hexo-cli

Github-Pages 设置

  • 新建仓库，命名为username.github.io

  • 在Settings - Pages - Build and deployment - Source中选择 Deploy from a branch

  • 为你的本地git生成rsa-key并上传到Github

Hexo创建博客

  • 博客文件夹初始化

    hexo init blog

    首先初始化一个博客项目，此处blog可以换成自己想要起的名称。该操作之后在当前目录下会出现一个叫做blog的新的文件夹

    cd blog

    进入blog文件夹下

    安装并设置icarus主题

    浏览器中输入localhost:4000查看网站情况

  • 博客写作

    • hexo new <layout> name

      新建一个模板为layout，名称为name的文章。layout通常为post。

      文章为MarkDown格式，icarus默认支持KaTeX和MatJax两种latex公式渲染。

    • 可以为文章添加FrontMatter属性：

      其中cover和thumbnail的图片路径从source开始。cover图片路径为blog/source/cover/XXX.png时候，FrontMatter属性为cover: cover/XXX.png。

    • 另外还可以自定义修改网站样式，修改_config.icarus.yml中参数，具体可以参考Hexo-Icarus官方网站教程。

    • 还可以直接修改icarus源代码自定义样式和功能，如栏宽，目录锁定等等。具体可以参考博客

Github-Pages部署

  • 安装hexo-deployer

    npm install hexo-deployer-git --save

  • 在blog\_config.yml中修改deployer部分代码

  • 执行hexo clean && hexo deploy

  • 浏览username.github.io检查网站是否正常运作

设置私人域名

  • github repo页面 Settings - Pages - Custom domain 设置你的个人域名

  • 域名管理网站DNS页面，增加CNAME项目，解析到username.github.io

  • 为了防止每次hexo deploy时丢失CNAME，可以在blog/source/目录下创建一个名称为CNAME的文件，将个人域名填入。
